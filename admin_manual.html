Roles & Permissions

Admin Role System

Admin (Full Access)

  • Access to ALL cities, ALL tabs, ALL reports.
  • Can manage roles — grant or revoke Admin and Site Admin.
  • Can access Settings and Page Settings tabs.
  • Can view and manage the complete audit log.
  • Only one or a few trusted people should hold this role.
All Cities All Reports Settings Role Manager

Site Admin (City-Scoped)

  • Access to ASSIGNED cities only — sees only their location's data.
  • Cannot access: Settings, Page Settings, Role Manager.
  • Report access controlled by the Admin via SiteAdminPermissionsCard.
  • City selection is locked to their assigned cities.
  • Ideal for city managers and facility operators.
Assigned Cities No Settings No Role Manager

First-Time Setup

  1. An existing admin sets up your account at /admin/setup
  2. You'll need the admin setup password — ask your system admin
  3. Enter the password to claim the Admin role
  4. You'll be redirected to the admin panel automatically
  5. Change your password immediately in Settings
Setup URL
/admin/setup

Day-to-Day Access

  • Admins are auto-redirected to /admin on login
  • The sidebar shows all available sections for your role
  • The City Selector (top of sidebar) filters all data
  • Admins see "All Cities" option; site admins see only their assigned cities
  • SessionStorage remembers your last selected city across page reloads
Bookmark /admin for quick access.
The sidebar collapses on smaller screens — use the hamburger icon.
If you ever lose admin access, ask a co-admin to reinstate your role via Role Manager → Settings tab.

Creating a Walk-in Booking

  1. Go to Walk-in Booking tab in the sidebar
  2. Select city and bay from the dropdowns
  3. Choose date and an available time slot
  4. Select session type — Practice or Coaching
  5. Enter customer details (member lookup or guest)
  6. Choose payment method — see options on the right
  7. Confirm — booking created, calendar event added automatically

Payment Options

  • Online: Generate a Razorpay payment link for the customer
  • Cash: Mark as cash payment — recorded immediately
  • Card (POS): Mark as card terminal payment
  • Advance: Deduct from customer's advance account balance
An invoice is automatically generated for every paid walk-in session. It appears immediately in Sales & Invoices.
Auto-Invoice Calendar Sync
Coaching walk-ins created by admin are auto-approved — admin-created bookings skip the member approval queue.

Loyalty · EDGE Rewards

EDGE Rewards Programme

The EDGE Rewards tab has 7 sub-tabs covering the full lifecycle of the loyalty programme — from earning rules to redemption to gifting.

Configuration Sub-Tabs

  • Catalogue: Create/edit redeemable rewards — name, description, point cost, availability, daily redemption cap, usage gate %
  • Rules: Set earning rules per event type — walkin, coaching, practice, renewal. Base points per unit.
  • Multipliers: Bonus multipliers — off-peak (auto from bay config), coaching, first-visit, tier-based (eagle/birdie), renewal
  • Milestones: Bonus points at hours-logged thresholds — e.g. 10 hrs = 500 pts, 50 hrs = 2,000 pts
Earning Rules Multipliers Milestones Catalogue

Activity Sub-Tabs

  • Transactions: Full log of all point earn and redemption events across all users — filterable by user, type, date
  • Gifts: Send manual rewards to specific users; view auto-gift rule outcomes and delivery status
  • Auto-Gift Rules: Configure automatic gifts triggered by events — e.g. signup, first booking, tier upgrade
Auto-gifts run on trigger events — a signup gift, for example, is sent immediately when a new user creates an account.
Transaction Log Manual Gifts Auto-Gifts
Points are only clawed back for cancelled bookings — rewards that have already been redeemed by a member are considered final and are not reversed.

Reports

Reports & Analytics

Available Reports

  • Revenue Tab: Revenue transactions, breakdown by type — bookings, packages, shop, walk-ins — with date range and city filters
  • Expense Reports: Expenses by category and vendor with period comparison
  • P&L View: Revenue vs Expenses for any period — net profit or loss
  • Product Profitability: Revenue vs COGS per product where cost is configured
  • GSTR-1: Tax return data export for the selected month — HSN summary, invoice list, B2B/B2C split
  • Advance Accounts: Pending advance balances per member

Access Control for Reports

  • Full Admin: Access to all report tabs without restriction
  • Site Admin: Report tab visibility is individually controlled by the Admin via Settings → Site Admin Permissions card
  • Admin can toggle visibility of each report tab independently per site admin
  • A site admin with no reports enabled will not see the Reports section at all
Report Exports
CSV Download GSTR-1 JSON Date Range Filter
All reports respect the city filter — switch cities to see location-specific financials.
GSTR-1 export is in JSON format compatible with the GST portal. Always verify with your CA before filing — the system generates the data, not legal advice.

Admin Only · Roles

Managing User Roles

Granting Roles

  • Navigate to Settings → Role Manager (Admin only)
  • To grant Admin: enter user's email, select Admin, click Grant
  • To grant Site Admin: enter email, select Site Admin, then assign the cities they should manage
  • City assignments are updated atomically — no partial state possible
After granting Site Admin, configure their report visibility in Settings → Site Admin Permissions Card so they see only the reports they need.

Revoking Roles

  • Click Revoke next to any user in the Role Manager list
  • You cannot revoke your own Admin role — safety check prevents accidental lockout
  • Revoking Site Admin also removes all their city assignments
  • All role changes are logged in the audit_log with timestamp and acting admin
Rate Limiting
5 role change attempts per 15 minutes per IP/user
Admin Only Audit Logged Atomic
Be careful granting the Admin role — admins have full system access including financial data, user management, and security settings. Prefer Site Admin for facility operators who only need city-level access.

Security

Security & Audit Log

Audit Log

Every change to critical data is automatically recorded and immutable.

Audited Tables
user_roles payment_gateways bay_pricing admin_config booking status invoices
  • Each entry records: who changed it, what table, what record, old value, new value, timestamp
  • Accessible via Admin → Settings → Audit Log viewer
  • Immutable — no one can edit or delete audit entries, including admins

Background Job Failures

  • Async jobs (emails, loyalty points, calendar events) are continuously monitored
  • Any failure is logged to the background_job_failures table
  • Admins can view all unresolved failures in the Email Logs tab
  • Mark failures as resolved once investigated — keeps the queue clean
  • This ensures silent errors (e.g. a failed booking confirmation email) are never lost
Security Protections
  • Setup and password endpoints: 5 attempts / 15 min rate limit per IP
  • Passwords bcrypt-hashed — never stored in plain text
  • API secrets stored as environment variables, not in the database
Rate limiting protects the setup-admin and change-admin-password endpoints — 5 attempts per 15 minutes per IP. Exceeding this limit locks the endpoint temporarily.
